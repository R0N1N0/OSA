Nmap

¿Qué es Nmap?

Nmap o Network Mapper es una de las herramientas más utilizadas en el mundo del hacking, ya que permite realizar exploraciones de redes y auditorías de seguridad. Nmap es una herramienta de código abierto creada por el equipo del mismo nombre. Está escrita en su mayoría por lenguaje C, C++ y Lua (https://nmap.org/). Su funcionalidad más interesante es la del escaneo masivo de dispositivos en una red, junto a los servicios, sistemas operativos y cortafuegos que utilizan. Nmap también cuenta con decenas de opciones para personalizar el escaneo de dispositivos ( -A para detectar versiones, sistemas operativos, puertos, etc. -sL para detectar objetivos…).

Metasploit

¿Qué es Metasploit?

Metasploit Framework o Metasploit para abreviar, es una herramienta muy popular en el mundo del hacking, pues permite automatizar y agilizar el pentesting (tests de intrusión) en todo tipo de servicios y dispositivos. Esta herramienta, por lo tanto, puede ayudar al descubrimiento de vulnerabilidades y a sus posibles vías de explotación. Este proyecto fue creado en 2003 por HD Moore utilizando el lenguaje de programación Perl, aunque en 2007 fue reescrito en Ruby y posteriormente adquirido por Rapid7 (https://www.metasploit.com/). Metasploit Framework consta de cuatro partes principales: Las interfaces, las bibliotecas, los módulos y los añadidos.

- Las interfaces son la manera de acceder al entorno de Metasploit. La más empleada y que vamos a utilizar más adelante es MSFconsole (uso de la consola).

- Las bibliotecas o APIs son conjuntos de instrucciones que dan forma a la sintaxis de Metasploit. Se utilizan tres: MSFCore, MSF y REX.

- Los añadidos o plugins son creados por la comunidad y sirven para extender las capacidades de Metasploit. Algunos ejemplos son nexpose o nessus para poder integrar estos buscadores de vulnerabilidades con Metasploit, lab para controlar distintas máquinas virtuales, etc.

- Los módulos son otra parte esencial de Metasploit, y definen el ámbito de la tarea que se quiere realizar. Hay de 6 tipos:

- Exploits: Estos son comandos o vectores de entrada que permiten acceder al sistema o aplicación de destino, aprovechándose de vulnerabilidades.
- Payloads: Estas son las cargas maliciosas de nuestra intrusión. Son fragmentos de código que se ejecutarán en la máquina o servicio atacado después del exploit.
- Post Explotación: Permiten realizar pruebas adicionales después de ya tener acceso al objetivo.
- Encoders: Cifran el contenido malicioso para intentar evadir cortafuegos y sistemas de detección y prevención de intrusiones.
- Auxiliaries: Son tipos de módulos sin una categoría concreta.
- NOPs: Generan secuencias de bytes aleatorias, para confundir a los IDS e IPS.

SQL Injection

¿Qué es SQL Injection?

SQL Injection es una vulnerabilidad de código que permite a un atacante acceder y/o modificar el contenido de una o varias bases de datos. Una inyección exitosa permite no solo realizar consultas, sino añadir, modificar, actualizar o borrar entradas en una base de datos relacional. Todos los ataques de SQL Injection se hacen a partir de una consulta SQL, pero de manera malintencionada. Las SQL Injection pueden dividirse en varios tipos:

- Basadas en errores: Este tipo se basa en los mensajes de error generados al intentar hacer la consulta. A partir de ahí se pueden deducir algunos datos o la estructura de la base de datos.

- Basadas en uniones: Se caracterizan por utilizar la palabra clave resevada UNION en la consulta original para inyectar o incrustar otra consulta malintencionada.

- Basadas en booleanos: Son las que se pueden comparar con valores booleanos (verdadero o falso), para comprovar si la consulta maliciosa funciona o incluso utilizarlas de cabecera para que se ejecuten.

- Basadas en tiempo: Se pueden inyectar consultas que fuerzen a la base de datos a acceder a muchas tablas, provocando así que tarden mucho tiempo. Suelen estar relacionadas con el siguiente tipo.

- Inyecciones ciegas: Son las inyecciones que no muestran resultado alguno, pero que en realidad, se están realizando. Eso quiere decir que aunque no se muestren resultados por pantalla, se siguen haciendo. Se combinan muy bien con las basadas en tiempo, ya que si pasa mucho tiempo antes de que la aplicación responda de nuevo, sabremos que hay una vulnerabilidad de SQLi.

- Basadas en consultas apiladas: Este tipo permite a un atacante poder hacer varias consultas en una sola solicitud, haciendo posible la inyección de muchas consultas malintencionadas.

- Inyecciones de segundo orden: Son las que se almacenan en la propia base de datos de forma latente, hasta que el atacante decida ejecutarlas.

Para el ejemplo de SQL Inyection, estaremos utilizando una Aplicación Web J*didamente Vulnerable (DVWA), la cual se puede encontrar en el siguiente enlace https://github.com/digininja/DVWA/pkgs/container/dvwa. También utilizaremos SQLMap, para ilustrar una posible manera de automatizar las consultas maliciosas.

Fuzzing Web

¿Qué es Fuzzing Web?

El Fuzzing Web es una técnica de pruebas de seguridad que se utiliza para identificar vulnerabilidades y errores en aplicaciones web. Consiste en enviar datos aleatorios, o de un diccionario para provocar respuestas inesperadas como SQL Injections, Buffer Overflows, Enumeración de archivos, etc.

Para automatizar el fuzzing web, existen una serie de programas como wfuzz o gobuster, los cuales prueban respuestas por parte del servidor a nivel de protocolo (HTTP GET/POST...) o a nivel de datos (HTML, JPG, TXT...) los valores que prueban pueden ser de varios tipos:

- Numeros: pueden ser enteros, flotantes, dobles, con signo o sin signo. Si se prueban numeros muy grandes o inesperados, se puede provocar un Buffer Overflow, que puede revelar posiciones de memoria y datos comprometidos.

- Caracteres: pueden ser saltos de linea, escapados, palabras reservadas o instrucciones, por ejemplo un & para incrustar una variable GET, o un ' para un SQL Injection.

- Secuencias binarias: son cadenas aleatorias de unos y ceros que pueden alterar el funcionamiento de la aplicación web.

- Metadatos: en el caso de no hacer fuzzing web, se pueden probar metadatos corruptos para ciertas extensiones de archivo, por ejemplo, cambiando el autor de una canción a un valor inválido en mp3 con id3.

Ejemplos

Vamos a revelar todos los secretos de una página web, en nuestro caso, utilizaremos de ejemplo la máquina CTF de Please Hack Me, ya que contiene una página web muy interesante con diversos directorios a explorar.

Página de inicio de HackMePlease.

Ahora que tenemos la página, podemos utilizar Gobuster para obtener todos los archivos y directorios expuestos. Primero, ponemos gobuster seguido de la opción dir, ya que queremos listar los directorios. También especificamos con -u la url de la página; en este caso con la IP ya basta. También debemos especificar un diccionario con -w para poder encontrar el contenido web. Un diccionario básico para hacer fuzzing web se puede encontrar en Web-Content/common.txt Finalmente, si queremos filtrar por extensiones de archivo, ponemos -x seguido de las extensiones que queremos, separadas por comas.

En este caso, vamos a buscar en la página 10.10.10.69 con el diccionario de palabras common.txt directorios y archivos con las extensiones txt,pdf,tar,gz,zip,js y php.

Una vez ejecutado podremos ver los resultados. Si pone Status:403 significará que ha encontrado el archivo o directorio, pero que no tiene acceso a él. Esto puede ser debido a que tiene algún tipo de protección como un .htaccess, por ejemplo. Si pone Status:200 significa que ha encontrado el archivo, y tiene acceso a él. Con el código Status:301 sabemos que el enlace que se proporciona redirige a otro enlace, ya que con ése no hay soporte. Finalmente, cabe comentar que por defecto gobuster no muestra los Status:404, ya que no son enlaces válidos, y por lo tanto, no nos interesan.

Se han encontrado carpetas con css, img, js, fonts y un index.html con la página principal.

Ahora que sabemos que tienen expuesta una carpeta con scripts de js, podemos seguir investigando, cambiando el directorio de búsqueda.

Si cambiamos la página web a 10.10.10.69/js/ podemos ver más resultados.
